Contesta las siguientes preguntas:

1.- Qué son los Modelos de estimación?

Los modelos de estimación son herramientas utilizadas para predecir o calcular el valor de una variable desconocida a partir de datos conocidos. Se emplean en diversas áreas como la economía, la ingeniería, la estadística y la gestión de proyectos. Estos modelos pueden utilizar métodos matemáticos, algoritmos y técnicas estadísticas para realizar estimaciones precisas, ayudando a tomar decisiones informadas y a planificar de manera efectiva.

2.- Investiga sobre diferentes modelos de estimación utilizados en la industria del desarrollo de software, como el modelo COCOMO o el método de puntos de función. Compara y contrasta dos modelos de estimación y explica en qué situaciones serían más adecuados

En la industria del desarrollo de software, el modelo COCOMO y el método de Puntos de Función son dos enfoques comunes para la estimación de proyectos. El modelo COCOMO se basa en el tamaño del software medido en líneas de código y es adecuado para grandes proyectos con datos históricos disponibles. En contraste, los Puntos de Función miden la funcionalidad del software desde la perspectiva del usuario, siendo útil para proyectos donde se necesita una estimación independiente del lenguaje de programación y la tecnología utilizada.

3.- A que llamamos la estimación de un proyecto de software?

La estimación de un proyecto de software es el proceso de predecir la cantidad de esfuerzo, tiempo y recursos necesarios para completar un proyecto. Este proceso incluye la evaluación de factores como el tamaño del software, la complejidad del proyecto, las capacidades del equipo y las tecnologías utilizadas. La estimación es fundamental para la planificación, presupuestación y gestión de proyectos, ayudando a establecer expectativas realistas y a asegurar la viabilidad del proyecto.

4-Enumera y explica brevemente los principios esenciales para el desarrollo de sistemas, como la modularidad, la cohesión, el acoplamiento y la abstracción. Elige uno de los principios y da un ejemplo de cómo se podría aplicar en el desarrollo de un sistema

Los principios esenciales para el desarrollo de sistemas incluyen la modularidad, que divide un sistema en partes manejables; la cohesión, que asegura que las tareas de un módulo estén estrechamente relacionadas; el acoplamiento, que busca minimizar la interdependencia entre módulos; y la abstracción, que oculta detalles complejos exponiendo solo las características esenciales. Por ejemplo, aplicar la modularidad en un sistema de gestión de inventarios implicaría crear módulos independientes para la gestión de productos, el seguimiento de pedidos y la administración de proveedores, facilitando el desarrollo y mantenimiento del sistema.

5-Crea una lista de las partes interesadas (stakeholders) que podrían estar involucradas en un proyecto de desarrollo de software para una aplicación móvil. Para cada parte interesada, identifica sus roles, responsabilidades y posibles expectativas con respecto al proyecto.

Las partes interesadas en un proyecto de desarrollo de software para una aplicación móvil incluyen a los clientes/usuarios finales que proporcionan retroalimentación, los propietarios del producto que definen y priorizan los requisitos, los desarrolladores que escriben el código, los diseñadores de UX/UI que crean la interfaz, los gerentes de proyecto que supervisan el desarrollo, los testers/QA que aseguran la calidad, los stakeholders financieros que proporcionan recursos y el departamento de marketing que promociona la aplicación.

6.- Diseña un plan de proyecto en el que se definan las diferentes fases y actividades necesarias para desarrollar un sistema de gestión de inventario. Asocia cada actividad con su respectiva fase y establece las dependencias lógicas entre ellas

Para desarrollar un sistema de gestión de inventario, el plan de proyecto se divide en varias fases clave: 1) Planificación: definición de requisitos y alcance del sistema. 2) Análisis y Diseño: elaboración de especificaciones detalladas y diseño de la arquitectura del sistema. 3) Desarrollo: implementación del software según los diseños aprobados. 4) Pruebas: verificación y validación del sistema para asegurar su funcionalidad y fiabilidad. 5) Implementación: despliegue del sistema en el entorno de producción. Cada fase tiene dependencias lógicas, donde el inicio de cada una depende de la finalización exitosa de la fase anterior para garantizar la coherencia y la calidad del sistema final.

7.- Elabora un conjunto de normas y estándares que deben seguirse en el desarrollo de software para garantizar la consistencia en la codificación y la documentación. Proporciona ejemplos concretos de cómo se aplicarían estas normas en la práctica.

Para asegurar la consistencia en el desarrollo de software, se deben seguir normas y estándares como: 1) Convenciones de codificación: utilizar nombres descriptivos para variables y funciones (por ejemplo, camelCase o snake_case), establecer reglas para la estructura del código y el uso de comentarios. 2) Documentación: incluir documentación en línea con Javadoc o similar para funciones y clases, definir estándares para los comentarios de bloque que expliquen el propósito, entradas y salidas de las funciones. 3) Control de versiones: emplear sistemas como Git, con convenciones de ramificación y etiquetado claras. 4) Pruebas y revisión de código: implementar revisiones de código periódicas y pruebas unitarias automáticas para validar la funcionalidad y la calidad del código. Estos estándares aseguran que el código sea legible, mantenible y coherente durante todo el ciclo de desarrollo.
